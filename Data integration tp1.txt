================================================================================
Part 1: Data Profiling and Data Quality Analysis
================================================================================

OBJECTIVE
---------
Perform a complete profiling of the dataset and evaluate its data quality.
Quantify missingness, detect anomalies, and propose remediation strategies.

INSTRUCTIONS
------------
You will work with the dataset provided (extracted from data.gouv.fr). Answer 
all guided questions clearly and justify your observations.

GUIDED QUESTIONS
----------------

A. Dataset Structure
--------------------
1. Column inventory: List all columns and their data types.
2. Semantic meaning: Explain the meaning and purpose of each column based on 
   the description provided.

B. Missing Values and Completeness
----------------------------------
1. Missing percentages: Compute the percentage of missing values for each 
   column.
2. Critical missingness: Identify which missing values are most problematic 
   for analysis and why.
3. Remediation strategies: Suggest appropriate actions.

C. Consistency and Validity Checks
----------------------------------
1. Value ranges: Identify out-of-range values or anomalies (Invalid 
   coordinates, negative ages).
2. Categorical anomalies: Detect unexpected or invalid categories.
3. Duplicates: Identify and quantify duplicate records.

D. Data Quality Summary
-----------------------
1. Quality report: Summarize the main issues discovered.
2. Impact analysis: Explain how these issues affect downstream analytics.

================================================================================
Part 2: Transformations, Modeling and Medallion Architecture
================================================================================

OBJECTIVE
---------
Design the transformation pipeline based on the profiling results, propose a 
simple analytical model, and produce a complete Medallion architecture diagram 
(Bronze -> Silver -> Gold).

A. Required Transformations (Silver Layer)
------------------------------------------
Based on the issues identified in Part 1, propose and justify the 
transformations needed:

1. Standardization: Normalize formats (dates, coordinates, categories).
2. Cleaning: Correct or remove invalid values.
3. Enrichment: Add derived fields (e.g., accident severity index, time-of-day 
   category).
4. Deduplication: Remove duplicate records.
5. Documentation: Document all transformation steps clearly.

B. Modeling (Gold Layer)
------------------------
Design a simple analytical model to support road-safety analysis.
Suggested structure with fact table (accidents) + dimensions as location, 
vehicle etc.

C. Medallion Architecture Diagram
---------------------------------
Produce a complete diagram showing:

- Bronze: raw open-data files
- Silver: cleaned, standardized, enriched tables
- Gold: analytical model (fact + dimensions)
- BI / dashboards: final consumption layer
